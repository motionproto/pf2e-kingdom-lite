Tertiary Heading - Outcome Headers

--type-body-large (16px / 1.14em)

Large body text for emphasized content. This size provides better readability for important information while maintaining the minimum 1em requirement.

--type-body (14px / 1em)

Standard body text used throughout the application. This is the base size that ensures all text meets the minimum 1em requirement for accessibility and readability.

Line Height Examples

--type-leading-tight (1.2)

Tight line height used for headings and titles where vertical space is limited. Best for single-line or short multi-line headings.

--type-leading-relaxed (1.5)

Relaxed line height provides comfortable reading for body text. This spacing reduces eye strain and improves readability for longer passages of text, making it ideal for descriptions and content areas.

Usage Guidelines

‚úÖ Best Practices

  • Always use semantic tokens (e.g., --type-heading-1-size) instead of raw font sizes
  • Maintain minimum 1em (14px) for all standard text
  • Use --type-badge-size (12px) only for small UI elements like badges
  • Combine size, weight, and line-height tokens for consistent typography
  • Use utility classes (.text-heading-primary) for rapid prototyping
